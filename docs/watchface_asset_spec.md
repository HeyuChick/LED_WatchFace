# LED 点阵表盘素材开发方案
## 技术规格 · 脚本生产指南

---

## 概览

本方案面向 Wear OS / Samsung Watch Face Studio (WFS) 表盘开发，目标是通过 Python 脚本（Pillow 库）批量生成以下素材：

- `bg_off_450.png`：熄灭状态灯珠背景
- `mask_450.png`：灯珠遮罩层
- `num_0.png ~ num_9.png`：主数字字体（逐字导入 WFS）
- `num_colon.png`、`num_dash.png`、`num_dot.png`、`num_percent.png`：主符号
- `label_A.png ~ label_Z.png`：辅助字母字体（短标签用）

所有资源共享同一套坐标系，确保在 WFS 中叠加时灯珠零错位。

---

## 第一章：坐标系与网格规格

### 1.1 基础常量

| 参数 | 值 | 说明 |
|---|---|---|
| 画布尺寸 | 450 × 450 px | WFS 推荐导出基准 |
| 灯珠矩阵 | 45 × 45 | 最大灯珠密度方案 |
| 单元节距 | 10 px | 450 ÷ 45 = 10，整除无浮点误差 |
| 灯珠中心偏移 | 5 px | 每单元中心 = 格索引 × 10 + 5 |

灯珠中心点公式（i, j 均从 0 起）：

```
cx = i * 10 + 5
cy = j * 10 + 5
```

45 × 45 共 2025 个灯珠，全部中心点均为整数，不存在半像素错位。

### 1.2 安全显示区

圆形表盘四角不可见，建议把灯珠矩阵分为三层：

| 区域 | 规格 | 用途 |
|---|---|---|
| 核心信息区 | 中心圆 r = 160 px（约 33 × 33 格） | 时间、日期、主信息 |
| 次级信息区 | r = 160 ~ 200 px | 电量、步数、心率标签 |
| 边缘装饰区 | r > 200 px（最外 5 格） | 仅做背景氛围，不放文字 |

圆形遮罩判断条件：

```python
dx = cx - 225  # 到画布中心距离
dy = cy - 225
in_circle = (dx**2 + dy**2) <= 225**2  # 225 = 表盘半径
```

## 第二章：灯珠参数规格

### 2.1 熄灭灯珠（用于背景层）

| 参数 | 值 |
|---|---|
| 形状 | 圆形（抗锯齿） |
| 外圈直径 | 7 px（半径 3.5 px） |
| 内芯直径 | 5.2 px（半径 2.6 px） |
| 外圈颜色 | `#303030`（较浅灰，模拟灯珠边缘） |
| 内芯颜色 | `#1A1A1A`（极暗灰，模拟关灯状态） |
| 背景颜色 | `#000000`（纯黑） |
| 绘制范围 | 覆盖完整 45 × 45 网格，包含图片四角 |

### 2.2 亮起灯珠（用于字体层）

| 参数 | 值 |
|---|---|
| 形状 | 圆形（抗锯齿） |
| 直径 | 7 px（半径 3.5 px） |
| 主体颜色 | 建议导出白色 `#FFFFFF`，在 WFS 中染色 |
| 背景 | 完全透明（PNG 透明通道） |

> **白色导出策略**：字符图做成白色透明底，在 WFS 的 Bitmap Font 颜色属性中设置为你想要的 LED 颜色（橙色、绿色、青色等），可以方便后期换色，不需要重导素材。

### 2.3 泛光层（可选，叠加在 mask 之上）

如需做 Glow 效果，另外输出一套字符图：

| 参数 | 值 |
|---|---|
| 灯珠直径 | 10~12 px，高斯模糊处理 |
| 透明度 | 50~70% |
| 混合模式 | 在 WFS 中设为 ADD |
| 画板尺寸 | 与对应字符画板一致（重要） |

## 第三章：字体规格

### 3.1 主数字字体（num_）

用于时间、日期数字、百分比。

| 参数 | 值 |
|---|---|
| 点阵逻辑 | 5 列 × 7 行 |
| 单元节距 | 10 px（继承全局网格） |
| 点阵区域 | 50 × 70 px |
| 左右内边距 | 各 5 px |
| 上下内边距 | 各 7 px |
| 单字符画板尺寸 | **60 × 84 px** |

字符画板尺寸推导：
- 宽：5 × 10 + 5 + 5（内边距）= 60 px
- 高：7 × 10 + 7 + 7（内边距）= 84 px

WFS 导入设置：
- Default Font Size = **84**
- 不在 WFS 内对字体做 scale

### 3.2 符号字体

| 符号 | 点阵 | 画板宽 | 画板高 | 文件名 |
|---|---|---|---|---|
| `:` 冒号 | 1 × 7 | 20 px | 84 px | `num_colon.png` |
| `-` 短横 | 3 × 7 | 40 px | 84 px | `num_dash.png` |
| `.` 小点 | 1 × 7 | 20 px | 84 px | `num_dot.png` |
| `%` 百分号 | 5 × 7 | 60 px | 84 px | `num_percent.png` |
| ` ` 空格 | — | 30 px | 84 px | `num_space.png`（全透明占位） |

> 冒号和小点宽度只有 20px，方便做 `HH:MM` 时中间间隙不过宽。

### 3.3 辅助字母字体（label_）

用于 `MON`、`HR`、`STEP`、`BATT` 等 2~4 字符短标签。

| 参数 | 值 |
|---|---|
| 点阵逻辑 | 4 列 × 5 行 |
| 单元节距 | 8 px（独立于全局 10px） |
| 左右内边距 | 各 4 px |
| 上下内边距 | 各 4 px |
| 单字符画板尺寸 | **40 × 48 px** |

WFS 导入设置：
- Default Font Size = **48**
- 不在 WFS 内对字体做 scale

> 辅助字体用 8px 节距而非 10px，是为了让小标签视觉更紧凑精致，与主数字之间形成层次对比。

## 第五章：脚本架构建议

### 5.1 配置参数集中管理

脚本顶部建议用一个配置字典统管所有参数，不要把数字散落在代码里：

```python
CONFIG = {
    # 全局
    "canvas_size": 450,
    "grid": 45,
    "cell": 10,

    # 灯珠
    "dot_radius_off_edge": 3.5,
    "dot_radius_off": 2.6,
    "dot_color_off_edge": (48, 48, 48),
    "dot_color_off": (26, 26, 26),
    "bg_color": (0, 0, 0),

    # 亮起灯珠（字符用）
    "dot_radius_on": 3.5,
    "dot_color_on": (255, 255, 255),

    # 主数字画板
    "num_cols": 5,
    "num_rows": 7,
    "num_cell": 10,
    "num_pad_x": 5,
    "num_pad_y": 7,
    # 画板自动算出: width=60, height=84

    # 辅助字母画板
    "lbl_cols": 4,
    "lbl_rows": 5,
    "lbl_cell": 8,
    "lbl_pad_x": 4,
    "lbl_pad_y": 4,
    # 画板自动算出: width=40, height=48

    # 输出
    "out_dir": "./assets_led",
}
```

### 5.2 核心绘制函数

建议把最底层的绘制逻辑拆成三个独立函数：

**draw_dot(draw, cx, cy, radius, color)**  
在指定中心点画一个抗锯齿圆形灯珠。由于 Pillow 的 `ellipse` 方法不支持原生抗锯齿，建议用超采样方案：以 4 倍尺寸绘制后再缩回原尺寸。

**draw_bg_off(config) → Image**  
生成完整的 450×450 背景图。遍历 45×45 格子，对每个格子计算中心点，在完整画布范围内绘制熄灭灯珠；每个熄灭灯珠先绘制较浅灰外圈，再绘制深灰内芯。

**draw_glyph(grid_data, dot_cell, dot_radius, pad_x, pad_y) → Image**  
接受一个二维 0/1 数组（点阵定义），返回透明背景的 PNG 图像。亮点（值为 1）绘制白色灯珠，暗点（值为 0）不绘制（保持透明）。

### 5.3 超采样抗锯齿实现

```python
from PIL import Image, ImageDraw

def draw_dot_aa(draw, cx, cy, r, color, scale=4):
    """在 draw 对象上以超采样方式画抗锯齿圆。
    实际操作：传入的坐标和半径需乘以 scale，
    最后图像整体缩回 1/scale 完成抗锯齿。
    建议在调用方创建 scale 倍大的画布，最后统一缩小。"""
    x0 = (cx - r) * scale
    y0 = (cy - r) * scale
    x1 = (cx + r) * scale
    y1 = (cy + r) * scale
    draw.ellipse([x0, y0, x1, y1], fill=color)
```

推荐的工作流程：
1. 创建 4 倍尺寸的画布（如 1800×1800）
2. 在大画布上绘制（坐标和半径全部 × 4）
3. 用 `Image.LANCZOS` 缩回目标尺寸（450×450）
4. 最终图像的圆形边缘会有自然的抗锯齿效果

### 5.4 mask 生成

mask 层与背景层共用同一套完整 45 × 45 灯珠中心点。  
区别仅在于：mask 以纯黑色为底，灯珠孔洞用 **alpha=0（完全透明）** 镂空。  
这样叠在字体层上方时，只有灯珠对应的圆孔是透明的，字体颜色可以透出来；其余黑色区域遮住字体的"溢出"部分。

mask 绘制逻辑：
1. 建立 RGBA 模式画布，底色为 `(0, 0, 0, 255)`（不透明黑）
2. 遍历 45 × 45 所有灯珠中心点，在对应位置用 `(0, 0, 0, 0)` 画圆（镂空）

### 5.6 SVG 附加输出

PNG 仍作为 WFS Bitmap Font 的主交付格式。SVG 作为附加产物，用于预览、编辑、版本审阅和兼容性尝试，不替代 PNG。

SVG 文件名与 PNG 一一对应，仅扩展名从 `.png` 改为 `.svg`；脚本会按文件类型分别输出到 `assets_led/png/` 和 `assets_led/svg/`：

- `bg_off_450.svg`
- `mask_450.svg`
- `num_0.svg ~ num_9.svg`
- `num_colon.svg`、`num_dash.svg`、`num_dot.svg`、`num_percent.svg`、`num_space.svg`
- `label_A.svg ~ label_Z.svg`

SVG 尺寸与对应 PNG 完全一致，并使用同尺寸 `viewBox`：

| 类型 | SVG 尺寸 | viewBox |
|---|---:|---|
| 背景 / 遮罩 | 450 × 450 | `0 0 450 450` |
| 主数字 | 60 × 84 | `0 0 60 84` |
| 冒号 / 小点 | 20 × 84 | `0 0 20 84` |
| 短横 | 40 × 84 | `0 0 40 84` |
| 百分号 | 60 × 84 | `0 0 60 84` |
| 空格 | 30 × 84 | `0 0 30 84` |
| 辅助字母 | 40 × 48 | `0 0 40 48` |

SVG 图形规则：

- 每个灯珠用 `<circle>` 表示。
- 字符 SVG 使用透明背景和白色圆点，便于后期染色或编辑。
- `num_space.svg` 只保留正确画板尺寸，不包含圆点。
- `bg_off_450.svg` 使用纯黑背景 `<rect>`，并为每个熄灭灯珠绘制浅灰外圈与深灰内芯两个 `<circle>`。
- `mask_450.svg` 可用 SVG `<mask>` 表达黑底圆孔，但实际 WFS 叠加优先使用 `mask_450.png`。

由于 WFS 对 SVG 作为 Bitmap Font 字符素材的兼容性存在不确定性，SVG 输出需要在 WFS 中实测后再作为正式导入格式使用。

## 第六章：WFS 组装规则

### 6.1 图层堆叠顺序（从底到顶）

| 层级 | 组件类型 | 资源文件 | 说明 |
|---|---|---|---|
| 1 | Image | `bg_off_450.png` | 熄灭灯珠背景，全程不动 |
| 2 | Bitmap Font | `num_*.png` 系列 | 主时间数字（亮灯） |
| 3 | Bitmap Font | `label_*.png` 系列 | 辅助短标签 |
| 4 | Image | `mask_450.png` | 遮罩层（修整字体边缘） |
| 5 | Bitmap Font (可选) | glow 版字体系列 | 泛光层，置于 mask 上方 |

### 6.2 Bitmap Font 导入规则

每个字符单独导入，逐行配置：

- 数字字体：Character = `0`，Image = `num_0.png`；…以此类推
- 符号字体：Character = `:`，Image = `num_colon.png`；等等
- 字母字体：Character = `A`，Image = `label_A.png`；…以此类推
- Default Font Size = **84**（数字字体）/ **48**（字母字体）
- WFS 内 Font Size 设为与 Default Font Size 相同
- **不在 WFS 内对 Bitmap Font 做任何缩放**

## 第七章：版式规划参考

### 7.1 主时间区

`HH:MM` 总像素宽度计算（使用主数字字体 60px + 冒号 20px）：

```
60 + 60 + 20 + 60 + 60 = 260 px
```

260px 居中放入 450px 画布时，两侧各留 95px 空白，符合核心信息区（r=160）约束。

建议垂直位置：画布中心略偏上，约 `y = 170~190`（中心线）。
